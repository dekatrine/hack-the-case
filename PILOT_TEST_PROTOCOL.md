# Пилотное тестирование Hack the Case

Этот пакет нужен для первого пользовательского исследования после деплоя.

## Цель

Проверить, помогает ли Hack the Case:

- лучше понимать логику решения бизнес-кейсов;
- структурировать ответ;
- формулировать более связные решения после использования сервиса.

## Выборка

- 5-10 участников;
- студенты 18-25 лет;
- желательно смешать тех, кто уже решал кейсы, и тех, кто почти не решал.

## Дизайн

Исследование проводится по схеме:

1. до использования сервиса;
2. работа с сервисом;
3. после использования сервиса.

## Шаги

### Шаг 1. Входное тестирование

Использовать:

- `research/PRE_TEST_FORM.md`
- `research/CASE_SKILL_SCORING.md`

Собрать:

- анкету респондента;
- короткий тест по методологии;
- мини-кейс без помощи сервиса.

### Шаг 2. Работа с сервисом

Дать участнику публичную ссылку на платформу.

Нужно зафиксировать:

- дошел ли до конца;
- сколько раз обращался к AI-Coach;
- какие блоки вызвали сложности;
- субъективно: что было понятно/непонятно.

### Шаг 3. Повторное тестирование

Использовать:

- `research/POST_TEST_FORM.md`
- `research/CASE_SKILL_SCORING.md`

Собрать:

- повторный тест по методологии;
- аналогичный по сложности мини-кейс;
- самооценку до/после.

### Шаг 4. Свод результатов

Заполнять:

- `research/PILOT_RESULTS_TEMPLATE.csv`

Считать:

- балл за методологический тест до и после;
- балл за мини-кейс до и после;
- прирост по каждому участнику;
- средний прирост по группе.

## Что сравнивать

Минимальный набор метрик:

- понимание постановки задачи;
- структура решения;
- глубина анализа;
- качество инициатив;
- корректность метрик;
- качество финальной рекомендации.

## Как интерпретировать

Первичный положительный эффект можно считать зафиксированным, если:

- у большинства участников есть рост по кейсовому баллу;
- участники лучше понимают логику этапов;
- итоговая обратная связь описывается как полезная;
- сервис не мешает прохождению сценария технически.
